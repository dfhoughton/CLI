Command Line Modifiers

The modifiers are constants in the dfh.cli.Cli.Mod enum. They modify the command line argument parser generated by the specification. Currently there are only two modifiers:

HELP

The HELP modifier causes the command line parser to add a help option to those listed in the specification. The names for this option will be taken from a preferred set. If none of these are available, names will be drawn from a backup set until one is found that is not already claimed. See dfh.cli.Cli in the API for the precise sets used.

THROW_EXCEPTION

This modifier is unlikely to be of any use outside unit testing. It causes the Cli object to throw an error where it would otherwise call System.exit() . In unit testing this is useful for examining usage output and confirming validation exceptions are thrown appropriately. It might be useful as well if you wanted to wrap what was otherwise a command line application in a service or some larger application.
